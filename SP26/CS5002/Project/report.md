# Project 1, Part 2 — Report

**Ryu Hemingway**

This project extends the gradient descent framework developed in Part 1 to a practical application: fitting a predictive model that estimates diastolic blood pressure from a patient's total cholesterol level. Two datasets were considered — one approximately linear and one intrinsically non-linear — and the suitability of different model classes was evaluated on each.

The code was organized as a pipeline. First, `load_data` reads a text file of (x, y) pairs into two NumPy arrays using `np.loadtxt`, returning cholesterol values as x and diastolic blood pressure values as y. Second, `cost_function(a, b, x, y)` computes the sum of squared residuals, g(a, b) = Σ (a·x⁽ⁱ⁾ + b − y⁽ⁱ⁾)², directly translating equation (3) from the assignment into a loop over all twenty patients. Third, the `gradient_descent_2d` function from Part 1 was reused without modification. It approximates the partial derivatives of g using a forward-difference scheme (∂g/∂a ≈ (g(a + h, b) − g(a, b)) / h with h = 10⁻⁷), updates both parameters simultaneously, and halts when the change in each falls below an epsilon tolerance. Passing g to `gradient_descent_2d` as an argument — exactly as was done in Part 1 for f1 and f2 — meant that no new optimizer logic had to be written.

For the first (linear) dataset, the raw values of x range from roughly 100 to 300 mmol/L and y ranges from roughly 59 to 99 mm Hg. Running gradient descent directly on these raw values was not straightforward. The issue is that the partial derivative with respect to a contains a factor of x⁽ⁱ⁾ inside the sum — analytically, ∂g/∂a = 2 Σ x⁽ⁱ⁾(a·x⁽ⁱ⁾ + b − y⁽ⁱ⁾) — while ∂g/∂b does not. Because x⁽ⁱ⁾ is one to two orders of magnitude larger than the residuals themselves, the gradient in the a direction ends up dramatically larger than the gradient in the b direction. A single learning rate α cannot serve both: an α large enough for b to make progress will cause a to oscillate and diverge, while an α small enough to keep a stable leaves b essentially stationary for thousands of iterations. Geometrically, the cost surface g(a, b) is an elongated narrow valley rather than a round bowl, and vanilla gradient descent navigates ill-conditioned surfaces of this kind poorly. Without scaling, it was not possible to find a set of initial parameters and hyperparameters for which the algorithm converged.

The fix suggested in the assignment hint was to standardize both variables before fitting. After computing μ_x, σ_x, μ_y, and σ_y, the scaled arrays x_s = (x − μ_x)/σ_x and y_s = (y − μ_y)/σ_y both have mean 0 and standard deviation 1. On scaled data, the cost surface is approximately round, so a single α works for both parameters, and gradient descent converges in a few hundred iterations starting from (0, 0). The scaled optima (a_s\*, b_s\*) were then converted back to the original-scale optima (a\*, b\*) analytically. Substituting the z-score definitions into y_s = a_s·x_s + b_s and solving for y in terms of x yields a\* = a_s·(σ_y/σ_x) and b\* = μ_y + b_s·σ_y − a_s·σ_y·μ_x/σ_x, which is what the code computes when converting back.

For the first dataset with α = 0.001 and epsilon = 10⁻⁸, the optimum was a\* ≈ 0.1998 and b\* ≈ 40.05 after 374 iterations, giving a final cost of g(a\*, b\*) ≈ 28.86. The resulting best-fit line y = 0.1998·x + 40.05 passes through the middle of the data cloud, and the average vertical deviation per point is about √(28.86 / 20) ≈ 1.2 mm Hg. The linear model therefore fits this dataset very well.

The second section reused the same pipeline against the new non-linear dataset. The optimum for the linear model was a\* ≈ 0.140, b\* ≈ 45.39 after 372 iterations, giving a cost of 179.23 — more than six times worse than in Section 1. Plotting the line against the data made the reason obvious. The actual relationship is flat-then-curved: blood pressure stays around 65 mm Hg for cholesterol values up to roughly 175 mmol/L and then rises steeply as cholesterol increases. No straight line can simultaneously hug a flat region and a steep region. The optimizer is doing its job and finding the best possible line; the problem is the model class f(x) = a·x + b, which is too rigid to represent a curved relationship.

The proposed modification was to enlarge the model class. A natural first choice that can capture a single curve is a quadratic, f(x) = a·x² + b·x + c. This requires minimizing g(a, b, c) = Σ (a·(x⁽ⁱ⁾)² + b·x⁽ⁱ⁾ + c − y⁽ⁱ⁾)², a function of three variables rather than two. The 2D optimizer from Part 1 was generalized to three dimensions in a straightforward way: a new `approx_partial` function selects which of a, b, or c to nudge, and `gradient_descent_3d` updates all three coefficients per iteration and checks that all three changes fall below epsilon before halting. The underlying algorithm — approximate the gradient, step against it, stop when updates become small — is otherwise identical. Standardization was applied in the same way, and the back-conversion formulas for the quadratic coefficients were derived by substituting x_s = (x − μ_x)/σ_x and y_s = (y − μ_y)/σ_y into y_s = a_s·x_s² + b_s·x_s + c_s and expanding to collect powers of x.

On the non-linear dataset, the quadratic optimum was A\* ≈ 0.000863, B\* ≈ −0.2052, C\* ≈ 76.73 after 1020 iterations, giving a cost of 18.67. Swapping the model class from linear to quadratic reduced the cost by nearly a factor of ten while using the same optimization algorithm and the same dataset. Visually, the resulting curve y = 0.000863·x² − 0.2052·x + 76.73 tracks the flat-then-rising shape of the data closely.

The main lesson from Part 2 is that the choice of f(x) matters more than the optimizer. Gradient descent finds the best parameters within whatever model class it is given; if that class cannot represent the true relationship in the data, no amount of further optimization will produce a good fit. Section 1 succeeded because the underlying data really was linear. Section 2 failed with the linear model and succeeded with the quadratic model because the quadratic class contains a function close to the data's actual shape. The same gradient descent framework was used throughout — only the hypothesis being fit changed.
